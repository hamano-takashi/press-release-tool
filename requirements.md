# プレスリリース原稿作成ツール 要件定義書

## 1. プロジェクト概要

### 1.1 プロジェクト名
プレスリリース原稿作成ツール（Press Release Generator）

### 1.2 目的
一般ユーザーが簡単にプレスリリース原稿を作成し、PDF・Word形式で出力できるWebアプリケーションを提供する。

### 1.3 対象ユーザー
- 一般ユーザー（マーケター、広報担当者、起業家、個人事業主など）
- プレスリリース作成の専門知識がなくても使用できる

## 2. 機能要件

### 2.1 必須機能

#### 2.1.1 原稿作成機能
- **入力項目（必須）**
  - タイトル（ヘッドライン）
  - ロゴ画像（アップロード）
  - メイン画像（アップロード）
  - その他画像（複数可、任意）
  - 本文（複数セクション対応）
  - 問い合わせ先情報
    - 担当者名
    - 電話番号
    - メールアドレス
  - リリース日（日付選択）

- **入力項目（任意）**
  - 会社名・組織名
  - スローガン・キャッチコピー
  - 背景セクション
  - 開発経緯セクション
  - その他セクション（自由追加）

#### 2.1.2 プレビュー機能
- リアルタイムプレビュー表示
- レスポンシブ対応（PC・タブレット・スマートフォン）
- 印刷プレビュー対応

#### 2.1.3 出力機能
- **PDF形式での出力**
  - A4サイズ対応
  - 複数ページ対応（コンテンツが増えても自動的にページ分割）
  - 高品質なレイアウト維持
- **Word形式での出力**
  - .docx形式
  - 編集可能な形式で出力

#### 2.1.4 フォーマット要件
画像に基づくフォーマット構造：

```
┌─────────────────────────────────────┐
│ 報道関係各位         2025年7月1日   │
│                    [会社名]          │
├─────────────────────────────────────┤
│                                     │
│  [メインタイトル/ヘッドライン]      │
│                                     │
│  [導入段落]                         │
│                                     │
│  [メイン画像 + スローガン]          │
│                                     │
│  ■背景                              │
│  [背景セクション]                   │
│                                     │
│  ■開発経緯                          │
│  [開発経緯セクション]               │
│                                     │
│  [その他セクション（任意）]         │
│                                     │
│  ＜問い合わせ先＞                   │
│  [担当者名]                         │
│  [電話番号]                         │
│  [メールアドレス]                   │
└─────────────────────────────────────┘
```

**レイアウト要件：**
- ヘッダー：左上に「報道関係各位」、右上に日付と会社名
- タイトル：大きく目立つフォント
- 画像：中央配置、適切なサイズ調整
- セクション：見出しは「■」マーク付き
- 本文：読みやすい行間・余白
- フッター：問い合わせ先情報
- **重要：コンテンツが増えても自動的にページ分割され、レイアウトが崩れない**

### 2.2 補助機能

#### 2.2.1 テンプレート機能
- 複数のテンプレートから選択可能
- カスタムテンプレート保存機能

#### 2.2.2 保存・読み込み機能
- 下書き保存（ローカルストレージまたはサーバー）
- 保存済み原稿の読み込み・編集

#### 2.2.3 画像管理機能
- 画像アップロード（ドラッグ&ドロップ対応）
- 画像のリサイズ・トリミング
- 画像の配置調整

### 2.3 AI支援機能

#### 2.3.1 ガイド付き作成機能（質問形式で導く機能）
初心者でも簡単にプレスリリースを作成できるよう、段階的な質問に答えることで完成させる機能。

**機能概要：**
- ステップバイステップの質問形式でユーザーを導く
- 各質問に答えるごとに、自動的に原稿が生成・更新される
- 画像のアップロードタイミングも適切に案内

**質問フロー例：**
1. **基本情報**
   - 会社名・組織名は？
   - 商品・サービス名は？
   - リリース日はいつですか？

2. **商品・サービス詳細**
   - 商品・サービスの特徴を教えてください
   - 主なターゲットは誰ですか？
   - 価格はいくらですか？

3. **背景・ストーリー**
   - なぜこの商品・サービスを作ったのですか？
   - 開発のきっかけや背景はありますか？
   - 現在の市場状況や社会課題との関連は？

4. **画像・ビジュアル**
   - ロゴ画像をアップロードしてください
   - メイン画像（商品写真など）をアップロードしてください
   - その他の画像があれば追加してください

5. **問い合わせ先**
   - 担当者名は？
   - 電話番号は？
   - メールアドレスは？

**機能詳細：**
- 質問は1つずつ表示（前の質問に答えると次へ進む）
- 進捗バーで全体の進捗を表示
- 前の質問に戻って修正可能
- 質問への回答に基づき、AIが自動的に原稿を生成
- 生成された原稿はプレビューで確認可能
- 生成後も手動で編集可能

#### 2.3.2 自動プレスリリース案生成機能
商品・サービスの情報を入力すると、リアルタイムのニュースや経済状況を分析し、メディアに取り上げられやすい複数のプレスリリース案を自動生成する機能。

**機能概要：**
- 商品・サービスの基本情報を入力
- AIが最新のニュース・トレンド・経済状況を分析
- メディアに取り上げられやすいアングルで複数の案を生成
- 各案の特徴や推奨理由を表示

**入力項目：**
- 商品・サービス名
- 商品・サービスの詳細説明
- 業界・カテゴリ
- ターゲット顧客
- 主な特徴・強み
- 価格帯
- リリース予定日

**生成プロセス：**
1. **情報収集・分析**
   - 最新のニュース・トレンドを収集（API連携）
   - 経済指標・市場動向を分析
   - 同業他社のプレスリリース事例を参照
   - メディアの関心が高いトピックを特定

2. **アングル提案**
   - 社会課題との関連性
   - トレンドとの関連性
   - 季節性・時事性
   - 業界動向との関連性
   - ユニークなストーリー性

3. **原稿生成**
   - 各アングルに基づいてプレスリリース原稿を生成（3〜5案）
   - タイトル、導入文、背景、開発経緯などを自動生成
   - メディアに取り上げられやすい構成を考慮

**出力形式：**
- 複数の案を一覧表示
- 各案に以下の情報を表示：
  - タイトル案
  - 主なアングル・特徴
  - 推奨理由（なぜこのアングルが良いか）
  - 想定されるメディア反応
  - 生成された原稿のプレビュー
- 気に入った案を選択して編集画面へ遷移
- 複数の案を組み合わせることも可能

**技術要件：**
- ニュースAPI連携（例：NewsAPI、Google News API）
- 経済データAPI連携（例：経済指標API）
- AI言語モデル（例：OpenAI GPT、Claude、Gemini）
- リアルタイムデータ更新（1日1回以上）

**注意事項：**
- 生成された原稿はあくまで案であり、ユーザーが必ず確認・編集する必要がある
- 事実確認が必要な情報は明示的に表示
- 著作権・商標権に配慮した生成

## 3. 非機能要件

### 3.1 パフォーマンス
- ページ読み込み時間：3秒以内
- プレビュー更新：リアルタイム（500ms以内）
- PDF/Word出力：10秒以内（通常サイズの場合）
- AI生成処理：30秒以内（通常の商品・サービス情報の場合）
- ガイド付き作成の質問応答：1秒以内（次の質問表示まで）

### 3.2 ユーザビリティ
- 直感的なUI/UX
- 操作ガイド・ヘルプ機能
- エラーメッセージは分かりやすい日本語で表示

### 3.3 互換性
- 主要ブラウザ対応（Chrome、Firefox、Safari、Edge）
- レスポンシブデザイン（PC・タブレット・スマートフォン）

### 3.4 セキュリティ
- アップロードファイルのサイズ制限（例：10MB/画像）
- ファイル形式チェック（画像：jpg, png, gif, webp）
- XSS対策

## 4. 技術要件

### 4.1 フロントエンド
- **推奨技術スタック**
  - React / Vue.js / Next.js（いずれか）
  - TypeScript
  - CSS Framework（Tailwind CSS / Material-UI など）
  - PDF生成ライブラリ（jsPDF / react-pdf / pdfmake など）
  - Word生成ライブラリ（docx / docxtemplater など）

### 4.2 バックエンド（必要に応じて）
- Node.js / Python / Ruby on Rails など
- ファイルストレージ（画像保存用）

### 4.3 AI・外部API連携
- **AI言語モデル**
  - OpenAI GPT-4 / GPT-3.5
  - Anthropic Claude
  - Google Gemini
  - または同等の日本語対応AIモデル
- **外部API**
  - ニュースAPI（NewsAPI、Google News API など）
  - 経済データAPI（必要に応じて）
  - リアルタイムデータ取得・分析機能

### 4.4 デプロイ
- 静的サイトホスティング（Vercel、Netlify、GitHub Pages など）
- またはサーバーレス（AWS Lambda、Cloud Functions など）
- AI機能はバックエンドAPI経由で実装（セキュリティ・コスト管理のため）

## 5. UI/UX要件

### 5.1 画面構成

#### 5.1.1 メイン画面（通常モード）
- 左側：入力フォームエリア
- 右側：プレビューエリア（リアルタイム更新）
- 上部：ツールバー（保存、出力、設定など）
- モード切替：通常モード / ガイドモード / AI生成モード

#### 5.1.2 ガイド付き作成画面
- 中央：現在の質問と回答フォーム
- 左側：進捗バーと質問リスト
- 右側：生成中の原稿プレビュー（リアルタイム更新）
- 下部：前へ/次へボタン、スキップ機能
- 上部：モード切替ボタン

#### 5.1.3 AI生成画面
- 上部：商品・サービス情報入力フォーム
- 中央：生成ボタンとローディング表示
- 下部：生成された案の一覧表示
  - カード形式で各案を表示
  - アングル、推奨理由、プレビューを表示
  - 選択ボタンで編集画面へ遷移

#### 5.1.4 入力フォーム
- セクションごとに折りたたみ可能
- 必須項目は明確に表示
- 入力支援（日付ピッカー、画像アップローダーなど）

#### 5.1.5 プレビュー
- 実際の出力に近い表示
- スクロール可能
- ページ区切り表示

### 5.2 デザイン原則
- シンプルで使いやすい
- プロフェッショナルな見た目
- アクセシビリティ配慮

## 6. 出力フォーマット詳細

### 6.1 PDF出力
- **ページ設定**
  - 用紙サイズ：A4（210mm × 297mm）
  - 余白：上下左右 20mm
  - フォント：日本語フォント（游ゴシック、メイリオなど）

- **レイアウト**
  - ヘッダー：固定位置
  - 本文：流動的（コンテンツに応じて自動改ページ）
  - 画像：最大幅を設定し、アスペクト比維持
  - ページ番号：フッター中央（2ページ目以降）

### 6.2 Word出力
- **形式**
  - .docx形式
  - 編集可能な状態で出力

- **レイアウト**
  - PDFと同様の構造
  - スタイル適用（見出し、本文など）

## 11. 追加機能詳細仕様

### 11.1 ガイド付き作成機能の詳細仕様

#### 11.1.1 質問カテゴリと質問例

**カテゴリ1：基本情報**

- Q3: リリース日を選択してください

**カテゴリ2：商品・サービス詳細**
- Q4: この商品・サービスの主な特徴を3つ教えてください

- Q6: 価格はいくらですか？（無料の場合は「無料」と入力）
- Q7: どこで購入・利用できますか？

**カテゴリ3：背景・ストーリー**
- Q8: なぜこの商品・サービスを作ったのですか？
- Q9: 開発のきっかけや背景を教えてください

**カテゴリ4：画像・ビジュアル**
- Q11: ロゴ画像をアップロードしてください
- Q12: メイン画像（商品写真など）をアップロードしてください
- Q13: その他の画像があれば追加してください（任意）

**カテゴリ5：問い合わせ先**
- Q14: 担当者名を教えてください
- Q15: 電話番号を教えてください
- Q16: メールアドレスを教えてください

#### 11.1.2 AI生成ロジック
- ユーザーの回答を収集
- 回答内容を分析し、プレスリリースの構成要素を抽出
- プレスリリースのベストプラクティスに基づいて原稿を生成
- タイトル、導入文、背景、開発経緯などを自動生成
- 生成された原稿をプレビューに反映

### 11.2 自動プレスリリース案生成機能の詳細仕様

#### 11.2.1 データ収集・分析プロセス

**ステップ1：情報収集**
- 商品・サービスのカテゴリを特定
- 関連する最新ニュースを収集（過去1ヶ月）
- 経済指標・市場動向を取得
- 同業他社のプレスリリース事例を収集

**ステップ2：トレンド分析**
- メディアで取り上げられやすいトピックを特定
- 季節性・時事性を考慮
- 社会課題との関連性を分析
- 業界動向との関連性を分析

**ステップ3：アングル提案**
以下の観点から複数のアングルを提案：
- **社会課題解決型**：社会課題の解決に貢献する角度
- **トレンド連動型**：現在のトレンドと関連付ける角度
- **季節性活用型**：季節や時事に合わせた角度
- **ユニークストーリー型**：独自性やストーリー性を強調する角度
- **業界革新型**：業界に革新をもたらす角度

#### 11.2.2 生成される案の構成
各案には以下が含まれる：
- **タイトル案**：メディアの関心を引くタイトル
- **導入文案**：最初の段落
- **背景セクション案**：なぜ今このタイミングか
- **開発経緯案**：開発のストーリー
- **アングル説明**：なぜこのアングルが良いか
- **推奨理由**：メディアに取り上げられやすい理由
- **想定メディア**：どのようなメディアに響くか

#### 11.2.3 技術実装の考慮事項
- API呼び出しのレート制限対策
- エラーハンドリング（API障害時の対応）
- キャッシュ機能（同じ商品・サービス情報の再生成を高速化）
- プライバシー配慮（ユーザー情報の適切な管理）
